# LIS Skins Profit Calculator

Userscript для Tampermonkey, который помогает искать потенциально выгодные предметы на маркетплейсе LIS-SKINS и сравнивать их с ценами заявок на покупку в Steam Community Market.

## Что делает скрипт

- Загружает несколько страниц маркетплейса LIS-SKINS.
- Фильтрует карточки по минимальной скидке.
- Получает цены заявок на покупку из Steam.
- Считает профит с учетом комиссии Steam.
- Показывает профит в рублях и процентах прямо на карточке.
- Сортирует карточки по проценту выгоды.
- Показывает подробную таблицу расчета при наведении на плашку цены.
- Позволяет отменить текущую загрузку и сортирует уже полученные данные.

## Установка

1. Установите расширение Tampermonkey.
2. Создайте новый userscript.
3. Скопируйте содержимое файла `src.js`.
4. Сохраните скрипт.
5. Откройте страницу маркета LIS-SKINS.

Скрипт работает на страницах вида:

```text
https://lis-skins.com/*/market/*
```

Панель появляется только на страницах со списком карточек маркета.

## Настройки

- **Мин. скидка (%)** - минимальная скидка карточки для попадания в поиск. Если значение `0`, карточки не скрываются по скидке.
- **Страниц собрать** - сколько страниц LIS-SKINS загрузить перед расчетом.
- **Воркеров Steam** - количество параллельных запросов к Steam. Большие значения повышают риск временной блокировки.
- **Лотов в подсказке** - сколько строк заявок Steam показать в таблице при наведении на плашку цены.

Все настройки сохраняются в `localStorage`.

## Цвета плашек

- Зеленый - профит больше 20%.
- Желтый - профит от 0% до 20%.
- Красный - отрицательный профит.
- Сине-серый - нет заявок на покупку.
- Синий - пауза из-за временной блокировки Steam.

## Расчет профита

Скрипт считает профит с учетом примерной комиссии Steam:

```text
сумма зачисления = цена продажи / 1.15
профит = сумма зачисления - цена покупки на LIS-SKINS
```

Это приближенный расчет. Steam может округлять комиссии, а итоговая комиссия зависит от игры и цены предмета.

## Steam-лимиты

Если Steam возвращает временную блокировку, скрипт ставит глобальную паузу для всех запросов Steam в текущем поиске. Каждый следующий retry в рамках одного поиска увеличивает ожидание на 30 секунд.
