# Profit-Calculator

Userscript для поиска профитных предметов на LIS-SKINS с расчетом выгоды по заявкам Steam.

## Что делает

- Загружает несколько страниц маркетплейса LIS-SKINS.
- Фильтрует карточки по минимальной скидке.
- Получает цены заявок на покупку из Steam.
- Считает профит с учетом комиссии Steam.
- Показывает профит в рублях и процентах прямо на карточке.
- Сортирует карточки по проценту выгоды.
- Показывает подробную таблицу расчета при наведении на плашку цены.
- Позволяет отменить текущую загрузку и сортирует уже полученные данные.

## Установка

1. Установите расширение Tampermonkey.
2. Создайте новый userscript.
3. Скопируйте содержимое файла `src.js`.
4. Сохраните скрипт.
5. Откройте страницу маркета LIS-SKINS.

Скрипт работает на страницах вида:

```text
https://lis-skins.com/*/market/*
```

Панель появляется только на страницах со списком карточек маркета.

## Настройки

- **Скидка, от %** - минимальная скидка на LIS. При `0` показываются все карточки.
- **Страниц LIS** - сколько страниц LIS-SKINS загрузить для поиска.
- **Запросов Steam** - сколько запросов к Steam делать одновременно. Высокое значение повышает риск блокировки.
- **Строк в таблице** - сколько заявок Steam показать в таблице при наведении на плашку.

Все настройки сохраняются в `localStorage`.

## Цвета

- Зеленый - профит больше 30%.
- Изумрудный - профит от 20% до 30%.
- Янтарный - профит от 0% до 20%.
- Красный - отрицательный профит или ошибка расчета.
- Серый - нет заявок на покупку.
- Синий - загрузка цен Steam.
- Фиолетовый - пауза из-за временной блокировки Steam.

## Расчет

Скрипт считает профит с учетом комиссии Steam:

```text
сумма зачисления = максимальная сумма продавца, при которой сумма продавца + комиссии Steam не превышает цену продажи
профит = сумма зачисления - цена покупки на LIS-SKINS
```

Комиссия считается в копейках: 5% Steam + 10% комиссии игры с минимальной комиссией 1 копейка на каждую часть.

## Лимиты Steam

Если Steam возвращает временную блокировку, скрипт ставит глобальную паузу для всех запросов Steam в текущем поиске. Каждый следующий retry в рамках одного поиска увеличивает ожидание на 30 секунд.
