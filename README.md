# lis-skins-profit-calculator

Userscript для LIS-SKINS, который ищет потенциально выгодные предметы и считает прибыль относительно заявок на покупку в Steam.

## Возможности

- Загружает дополнительные страницы LIS-SKINS прямо из панели.
- Фильтрует карточки по минимальной скидке.
- Получает заявки Steam для CS2, Dota 2 и Rust через новый `/market/orderbook`.
- Для остальных случаев использует HTML/Steam fallback-парсеры.
- Учитывает категории CS2: `StatTrak`, `Souvenir`, exterior `FN/MW/FT/WW/BS`.
- Собирает корректные ссылки Steam для grouped market pages.
- Считает прибыль с учетом комиссии Steam.
- Показывает цену, количество заявок по лучшей цене, прибыль в рублях и процентах.
- Показывает подробную таблицу расчета при наведении на плашку.
- Сортирует карточки по выгоде.
- Позволяет отменить текущую загрузку.

## Установка

1. Установите Tampermonkey.
2. Создайте новый userscript.
3. Скопируйте содержимое `src.js`.
4. Сохраните скрипт.
5. Откройте маркет LIS-SKINS:

```text
https://lis-skins.com/*/market/*
```

Панель появляется на страницах со списком карточек маркета.

## Настройки

- **Скидка, от %** - минимальная скидка LIS. При `0` показываются все карточки.
- **Страниц LIS** - сколько страниц загрузить.
- **Потоков LIS** - сколько страниц LIS загружать одновременно.
- **Запросов Steam** - число параллельных запросов к Steam. Чем выше значение, тем выше риск временной блокировки.
- **Удалять ниже %** - при сортировке удалять карточки с выгодой ниже заданного порога.
- **Строк в таблице** - сколько строк заявок Steam показывать в tooltip-таблице.

Настройки сохраняются в `localStorage`.

## Как берутся цены Steam

Скрипт сначала использует новый endpoint Steam:

```text
GET https://steamcommunity.com/market/orderbook?q=Load&qp=[appid,"market hash name"]
Header: x-valve-request-type: queryAction
```

Steam возвращает компактную книгу заявок:

```json
{
  "success": true,
  "data": {
    "cBuyOrders": 5324,
    "rgCompactBuyOrders": [303147, 2, 302641, 1]
  }
}
```

`rgCompactBuyOrders` читается как пары:

```text
цена в копейках, количество
```

Если orderbook не сработал, скрипт пробует старые fallback-способы: парсинг HTML страницы Steam, render endpoint и histogram endpoint.

## Плашка и таблица

Плашка на карточке показывает:

```text
цена Steam [количество по лучшей цене] (прибыль, процент)
```

Общее количество заявок используется только когда Steam не отдает подробную таблицу, но на странице есть summary-текст. Если таблица найдена, в плашке показывается количество по первой строке таблицы.

Tooltip-таблица показывает расчет по нескольким ценам Steam:

- покупка на LIS
- продажа в Steam
- количество заявок
- прибыль без комиссии
- комиссия
- к получению
- итоговый профит
- процент выгоды

## CS2 категории

Скрипт распознает сокращения LIS:

```text
FN -> Factory New
MW -> Minimal Wear
FT -> Field-Tested
WW -> Well-Worn
BS -> Battle-Scarred
ST™ -> StatTrak™
```

Для grouped Steam market pages добавляется query-параметр exterior:

```text
Factory New     -> WearCategory0
Minimal Wear    -> WearCategory1
Field-Tested    -> WearCategory2
Well-Worn       -> WearCategory3
Battle-Scarred  -> WearCategory4
```

## Расчет прибыли

Скрипт считает итоговую сумму к получению после комиссии Steam:

```text
прибыль = сумма к получению после комиссии Steam - цена покупки на LIS
```

Комиссия считается в копейках:

```text
5% Steam + 10% комиссия игры
```

Каждая часть комиссии имеет минимум 1 копейку.

## Цвета плашек

- Зеленый - прибыль больше 30%.
- Изумрудный - прибыль от 20% до 30%.
- Янтарный - прибыль от 0% до 20%.
- Красный - отрицательная прибыль или ошибка.
- Серый - нет заявок на покупку.
- Синий - идет загрузка.
- Фиолетовый - пауза после ограничения Steam.

## Ограничения

- Steam может временно ограничивать частые запросы. В этом случае скрипт ставит глобальную паузу и повторяет запросы.
- Некоторые предметы могут отсутствовать в новом `/market/orderbook`; для них используется fallback.
- Корректность зависит от того, насколько LIS отдает название предмета, категорию и состояние в карточке.
